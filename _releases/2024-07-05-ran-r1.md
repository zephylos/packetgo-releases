## 개요

PacketGo RAN R1 릴리즈입니다. 무선 라우터 환경에서의 에이전트리스 단말 네트워크 접속 제어 기능을 제공합니다.

## 주요 기능 개발 내역

### 1. RAN 전용 Controller UI 개발

**DASHBOARD**
- RAN탭: 무선 라우터와 에이전트리스 디바이스의 현재 연결상태 확인

**OBJECT**
- 대리점 메뉴: 라우터를 사용하는 대리점을 추가하는 페이지 구현
- 고객 메뉴: 라우터를 사용하는 고객을 추가하는 페이지 구현
- 에이전트리스 디바이스 메뉴: 에이전트리스 단말을 등록할 수 있는 페이지 구현 (에이전트리스 단말 고정 ip 적용 가능)

**FLOW**
- 무선 서비스: 등록된 무선 라우터의 서비스 및 기능을 설정할 수 있는 페이지 구현 (무선라우터 고정 ip 대역 적용 가능)
- 무선 라우터: 무선 라우터의 정보를 입력하거나 볼 수 있는 페이지 구현
- 디바이스 플로우: 에이전트리스 단말의 네트워크 접속을 통제할 수 있는 플로우 기능 개발

### 2. 무선라우터 기능 개발

- 컨트롤러에서 허가된 port list 수신하여 모니터링 기능
- 라우터에 접속하는 에이전트리스 단말에 대한 IP, MAC 정보 관리 기능
- 정방향: 라우터와 에이전트리스 단말 간에 디바이스 플로우에 등록된 대로 통신이 가능하도록 개발
- 역방향: 에이전트 설치 단말에서 라우터와 연결된 에이전트리스 단말로 통신이 가능하도록 기능 개발

### 3. 컨트롤러 프로토콜 개발

- 컨트롤러와 무선 라우터가 사용할 수 있도록 개발
- 에이전트리스 단말 컨트롤러와 통신 가능하도록 개발

## 제약 사항

- 컨트롤러와 에이전트리스 단말의 연결 상태는 실시간이 아닌 최소 1~120초 정도 차이로 컨트롤러에서 모니터링됨
- 고정ip 할당 기능은 라우터를 reboot해야만 단말에 고정 ip 적용됨
- 라우터의 경우 프로세스 ID가 없기 때문에 동일한 데이터플로우를 인식할 수 밖에 없어서 접속완료 로그는 한번만 찍힘 (다시 로그 찍히는 시기는 라우터를 재연결하게되면 다시 로그 생성됨)

## 주의 사항

- 고정ip 할당 기능은 라우터를 reboot해야만 단말에 고정 ip 적용됨

## 참고 사항

- 무선라우터와 에이전트리스 단말 케이블 연결 시 DHCP로 ip할당됨
- 무선서비스 > 연결 디바이스 플로우가 기본 허용 ON일 경우 에이전트리스 단말 상태가 "대기" 상태에서도 디바이스 플로우 정책을 따름 (우선순위: 디바이스 플로우 기본 허용 ON > 에이전트리스 단말 상태)
- 구글의 경우는 UDP를 사용하고 있어서 TCP 데이터 패킷 제어는 되지 않음
- 역방향 시 사용되는 Agent파일은 기존 배포된 파일을 사용
