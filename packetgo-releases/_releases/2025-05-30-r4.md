## 개요

에이전트 종료 프로세스 로직 변경, 데이터 플로우 선 접속 후 인증 기능, 컨트롤러 보안 형상 설정, MFA 다중 매체 지원 등이 포함된 정규 릴리즈입니다.

> ※ Server(Controller/Gateway) 표준 제품 성능 개선 사항은 **6월 내 별도 릴리즈 예정**입니다.

## 주요 업데이트 내역

### 1. 에이전트 종료 프로세스 로직 변경

- 기존: 팝업 발생 시 앱 종료 처리
- 변경: 팝업 발생 시 로그아웃 처리 (단, "앱 종료"로 명시된 팝업은 기존처럼 종료)

### 2. 데이터 플로우 선 접속 후 인증 기능 개발

- 플로우 제어 영역에 "선 접속 후 인증 사용 여부" 기능 추가
- 애플리케이션 명칭 변경: 기존 "선접속 및 강화된 인증" → "접속 우선 허용 및 강화된 인증"
- 플로우 제어 영역을 선접속 후 인증 대역으로 설정할 경우 RDF 발생 전 패킷을 차단하지 않고 허용

### 3. 에이전트 공지 팝업 기능 추가

SYSTEM > 공지사항 관리에 등록된 공지사항은 로그인 후 에이전트에 표시

### 4. 컨트롤러 보안 형상 설정 기능 개발

- 공공 형상(CC보안 요구 사항)과 민간 형상 선택 가능
- 공공 형상: 패스워드 규칙 CC보안 요구 사항 기준, 관리자 계정 정책 관리자로만 추가 생성
- 민간 형상: 패스워드 규칙 민간 기준, 관리자 계정 시스템 관리자/정책 관리자 선택 가능

### 5. 에이전트 MFA 인증 시 다중 매체 지원

MFA 인증 시 인증 방식을 선택할 수 있도록 기능 구현

### 6. .net 재배포 패키지 포함

제품 설치 패키지에 .net 재배포 패키지 포함 (설치 시 재배포 패키지를 삭제 후 재 설치)

### 7. Windows 메모리 누수 증상 해결

v2.6.3.11 이하 버전(openssl 3.3.2)에서 발생되는 메모리 누수 증상 수정

### 8. 사용자 자동 로그인 기능 구현

자동 로그인 시 "간편 인증 사용" → "사용자 자동 로그인 접속 허용"으로 정책명 변경

### 9. 컨트롤러 VIP 대역 UI 설정 기능 추가

- 네트워크 경계 > SSL VPN 네트워크 정보에 "VIP대역" 항목 추가
- `server.conf`에서 ip 대역을 UI에서 직접 변경 가능하도록 개선

### 10. 에이전트 중복 접속 시 기존 접속 해제 기능 개발

- SYSTEM > 시스템 환경 설정 > 단말 정보에 옵션 추가
- 옵션 설정 시, 중복 접속 발생 시 기존 사용자 자동 로그아웃 처리 후 접속 가능

### 11. 관리자 설정에 따른 사용자 비밀번호 설정 기능 개발

SYSTEM > 컨트롤러 관리에 사용자 비밀번호 설정 방법 옵션 추가

### 12. 필수 설치 및 필수 실행 프로그램 검사 로직 개선

정책 설정 시 2가지 방식 지원: "설치 경로" + "프로그램명"

## 제약 사항

(Server 성능 관련 제약 사항은 6월 별도 릴리즈 노트 참고)
